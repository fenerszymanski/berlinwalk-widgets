# Blog Visual Plan

Last updated: 2026-05-15

Use this as the image checklist when moving today's drafts into Wix. Prefer Yusuf's own photos first, then official press/media-library images where usage is allowed, then reputable free-license images. Avoid generic stock shots, dark blurred hero images, or tourist-cliche images that do not show the actual subject.

## Global Rules

- Hero image: 16:9 or close, minimum 1600 px wide, bright enough for mobile crop.
- Inline images: 3:2 or 4:3, minimum 1200 px wide.
- File names: lowercase, descriptive, hyphenated, no spaces. Example: `berlin-september-museum-island-evening.jpg`.
- Compression target: under 250 KB for inline images, under 400 KB for hero if quality needs it.
- Alt text: describe the scene plainly and include the topic when natural. Do not stuff keywords.
- Do not place an image immediately before/after a widget if the page starts feeling heavy. Let widgets carry some visual rhythm.
- For event images, use only official press images, own photos, or images with clear reuse permission.

Notes:

## Asset Search Checklist

Priority order for each post:

1. Check Yusuf's own Berlin photo library.
2. Check official venue/event press pages for allowed media use.
3. Use Wikimedia Commons only when license and attribution are clear.
4. Use Unsplash/Pexels only for broad seasonal mood, not for specific museums/events.
5. Generate an AI image only for abstract seasonal filler, never for a real museum, event, or historical site where accuracy matters.

## Wix Upload Checklist

- Upload hero first and set it as post cover/social image.
- Add inline images only where they support a section, not as decoration.
- Fill alt text immediately while adding each image.
- Check mobile crop for hero image before saving the draft.
- After widgets are embedded, preview mobile and desktop to make sure image/widget spacing does not feel crowded.
